Funding account switching

Funding account switch (FAS) allows payroll administrators to change the financial account their payroll provider uses to fund their payroll. Citadel makes FAS easy to integrate into your system with a simple process. Click "Connect" below to start the FAS process. Monitor console logs to learn what's happening behind the scenes.

The FAS process is now complete. Here's what happened.

Initializing the Citadel Bridge

  1. The quickstart front end requested a bridge token from the quickstart back end.
  2. The quickstart back end requested a bridge token from Citadel, providing fas as the value for product_type and all the account information for the new bank account to fund the administrator payroll.
  3. Citadel provided the quickstart backend with a bridge token, which the quickstart back end returned to the quickstart front end
  4. The quickstart front end initialized the Citadel Bridge with the provided bridge_token

Starting the FAS process

  1. The user logged into their payroll provider through the Citadel Bridge
  2. The Citadel Bridge provided a public_token to the quickstart front end, who passed that public_token to the quickstart back end
  3. The quickstart back end exchanged that public_token with Citadel for an access_token and stored that access_token for later use.

Completing the FAS process

  1. The user provided the quickstart front end with the values of the two micro deposits the payroll provider sent to the new bank account
  2. The quickstart front end provided these micro deposit values to the quickstart back end
  3. The quickstart sent Citadel a Task refresh request providing the previously stored access_token and the two micro deposit values
  4. Citadel provided these two micro deposit values to the payroll provider, completing the Funding account switch process.
