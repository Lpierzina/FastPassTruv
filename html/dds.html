Direct deposit switching

Direct deposit switch (DDS) allows individuals to change the financial account their payroll provider uses to deposit their pay into. Citadel makes DDS easy to integrate into your system with a simple process. Click "Connect" below to start the DDS process. Monitor console logs to learn what's happening behind the scenes.

The DDS process is now complete. Here's what happened. What data was returned?

Initializing the Citadel Bridge

  1. The quickstart front end requested a bridge token from the quickstart back end.
  2. The quickstart back end requested a bridge token from Citadel, providing deposit_switch as the value for product_type and all the account information for the new bank account to fund the administrator payroll.
  3. Citadel provided the quickstart backend with a bridge token, which the quickstart back end returned to the quickstart front end.
  4. The quickstart front end initialized the Citadel Bridge with the provided bridge_token.

Starting the DDS process

  1. The user logged into their payroll provider through the Citadel Bridge.
  2. The Citadel Bridge provided a public_token to the quickstart front end, who passed that public_token to the quickstart back end.
  3. The quickstart back end exchanged that public_token with Citadel for an access_token and stored that access_token for later use.

Retrieving the DDS data

  1. The quickstart back end sent Citadel a request for DDS data providing the previously stored access_token.
  2. Citadel provided the data to the quickstart back end who sent it to the quickstart front end.
